# AI Hall of Shame (AHOS) ‚Äî Copilot Instructions

## Project Overview
A humorous SPA for crowd-sourcing AI misbehavior stories. Reddit-like forum with flat comments, passkey auth, voting. Deployed on Cloudflare Pages + Workers + D1 at $0/month.

## Tech Stack
- **Frontend**: SvelteKit 2 (Svelte 5 runes) SPA mode (`ssr: false`), Tailwind CSS v4
- **Backend**: SvelteKit API routes (`+server.ts`) running as Cloudflare Workers Functions
- **Database**: Cloudflare D1 (SQLite)
- **Auth**: WebAuthn passkeys (`@simplewebauthn/server` + `@simplewebauthn/browser`), JWT via `jose`
- **Tooling**: Node 24 (volta-pinned), pnpm via corepack, TypeScript 5.9 strict

## Architecture
- Single codebase, single deployment to Cloudflare Pages
- Pages (`src/routes/*.svelte`) are client-side rendered (SPA)
- API routes (`src/routes/api/**/+server.ts`) run server-side as Workers
- No cookies ‚Äî JWT in `Authorization: Bearer` header, stored in `sessionStorage`
- No SSR ‚Äî all page rendering is client-side
- D1 accessed via `platform.env.DB` in server routes

## Key Design Decisions
- **Plain text posts only** ‚Äî no markdown, no `{@html}` (XSS prevention)
- **Dark-only theme** ‚Äî "Neon Shame" palette (no theme toggle)
- **Flat comments only** ‚Äî no nested/threaded comments
- **Optimistic voting** ‚Äî instant UI update, background API call
- **7-day JWT expiry** ‚Äî stored in sessionStorage + Svelte store
- **Username generation** ‚Äî `SHA-256(credentialId)` ‚Üí adjective-noun-number

## Code Conventions
- Use Svelte 5 runes (`$state`, `$derived`, `$effect`) ‚Äî no legacy `$:` reactive statements
- Use `$props()` for component props ‚Äî no `export let`
- TypeScript strict mode everywhere
- All DB queries use parameterized prepared statements (`db.prepare(sql).bind(...)`) ‚Äî NEVER string interpolation
- Sort queries use whitelist mapping ‚Äî NEVER interpolate user input into SQL
- Input validation with Zod schemas in every API route
- Security headers set via `_headers` (Cloudflare Pages declarative header config ‚Äî lives at project root, NOT in `static/`)

## The `_headers` File

`_headers` is a [Cloudflare Pages feature](https://developers.cloudflare.com/pages/configuration/headers/) for setting HTTP response headers declaratively. It is NOT a custom file ‚Äî it's a standard mechanism, like `_redirects`. The `adapter-cloudflare` plugin includes it in the deployment automatically. It sets:
- `Content-Security-Policy`
- `X-Frame-Options`
- `X-Content-Type-Options`
- `Referrer-Policy`
- `Permissions-Policy`
- CORS headers for `/api/*` (allowing `https://hallofshame.cc`)

## Security Rules
1. **NEVER read, display, or output the contents of `.env` or any `.env.*` file** ‚Äî these contain live secrets. This is a hard stop, no exceptions.
2. Never use `{@html}` with user content
3. Always use `db.prepare().bind()` ‚Äî never template literals in SQL
4. Rate limit all POST endpoints (5/min per user, 10/min per IP)
5. Rate limit GET endpoints (60/min per IP)
6. Timing-safe comparison for backup code verification
7. Validate all inputs with Zod before processing
8. Check bans ‚Üí rate limits ‚Üí auth (in that order) in middleware

## Humor Guidelines
- Humor goes in copy/microcopy (empty states, tooltips, footer), NOT in validation errors or API error codes
- API error responses must be machine-readable JSON with optional human-friendly `message`
- Backup code modal is SERIOUS ‚Äî no jokes there
