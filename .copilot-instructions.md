# AI Hall of Shame (AHOS) — Copilot Instructions

## Project Overview
A humorous SPA for crowd-sourcing AI misbehavior stories. Reddit-like forum with flat comments, passkey auth, voting. Deployed on Cloudflare Pages + Workers + D1 at $0/month.

## Tech Stack
- **Frontend**: SvelteKit 2 (Svelte 5 runes) SPA mode (`ssr: false`), Tailwind CSS v4
- **Backend**: SvelteKit API routes (`+server.ts`) running as Cloudflare Workers Functions
- **Database**: Cloudflare D1 (SQLite)
- **Auth**: WebAuthn passkeys (`@simplewebauthn/server` + `@simplewebauthn/browser`), JWT via `jose`
- **Tooling**: Node 24 (volta-pinned), pnpm via corepack, TypeScript 5.9 strict

## Architecture
- Single codebase, single deployment to Cloudflare Pages
- Pages (`src/routes/*.svelte`) are client-side rendered (SPA)
- API routes (`src/routes/api/**/+server.ts`) run server-side as Workers
- No cookies — JWT in `Authorization: Bearer` header, stored in `sessionStorage`
- No SSR — all page rendering is client-side
- D1 accessed via `platform.env.DB` in server routes

## Key Design Decisions
- **Plain text posts only** — no markdown, no `{@html}` (XSS prevention)
- **Dark-only theme** — "Neon Shame" palette (no theme toggle)
- **Flat comments only** — no nested/threaded comments
- **Optimistic voting** — instant UI update, background API call
- **7-day JWT expiry** — stored in sessionStorage + Svelte store
- **Username generation** — `SHA-256(credentialId)` → adjective-noun-number

## Code Conventions
- Use Svelte 5 runes (`$state`, `$derived`, `$effect`) — no legacy `$:` reactive statements
- Use `$props()` for component props — no `export let`
- TypeScript strict mode everywhere
- All DB queries use parameterized prepared statements (`db.prepare(sql).bind(...)`) — NEVER string interpolation
- Sort queries use whitelist mapping — NEVER interpolate user input into SQL
- Input validation with Zod schemas in every API route
- Security headers set via `static/_headers`

## File Organization
```
src/
  lib/
    components/    # Svelte components
    server/        # Server-only code (auth, db, ratelimit)
    stores/        # Svelte stores (auth)
    types/         # TypeScript type definitions
    utils/         # Client utilities (api fetch wrapper, passkey helpers)
  routes/
    api/           # API endpoints (+server.ts files)
    post/[id]/     # Post detail page
    submit/        # New post page
```

## Security Rules
1. Never use `{@html}` with user content
2. Always use `db.prepare().bind()` — never template literals in SQL
3. Rate limit all POST endpoints (5/min per user, 10/min per IP)
4. Rate limit GET endpoints (60/min per IP)
5. Timing-safe comparison for backup code verification
6. Validate all inputs with Zod before processing
7. Check bans → rate limits → auth (in that order) in middleware

## Humor Guidelines
- Humor goes in copy/microcopy (empty states, tooltips, footer), NOT in validation errors or API error codes
- API error responses must be machine-readable JSON with optional human-friendly `message`
- Backup code modal is SERIOUS — no jokes there
